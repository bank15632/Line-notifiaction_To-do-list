## ภาพรวมระบบ

ระบบนี้เป็น **Web App สำหรับจัดการงาน (To-Do List)** ที่มี **LINE Bot แจ้งเตือน** อัตโนมัติ

โครงสร้างข้อมูล:
```
โปรเจค (Project)
  └── Task (งาน)
        └── Sub-task (งานย่อย)
```

แต่ละ Task/Sub-task จะมี:
- **สถานะ**: รอดำเนินการ (Todo) / กำลังทำ (Doing) / เสร็จแล้ว (Done)
- **Deadline**: วันกำหนดส่ง + แสดงจำนวนวันที่เหลือ (ติดลบหากเลยกำหนด)
- **การพึ่งพา**: สามารถตั้งให้ "รองานจาก Task/Sub-task อื่น" ได้

## การใช้งาน Web App

### 1. จัดการโปรเจค

#### สร้างโปรเจคใหม่
1. คลิก **"+ สร้างโปรเจคใหม่"** ที่หน้าแรก หรือหน้าโปรเจคทั้งหมด
2. กรอก **ชื่อโปรเจค** (จำเป็น) และ **รายละเอียด** (ไม่จำเป็น)
3. กด **"สร้างโปรเจค"**

#### ดูภาพรวมโปรเจค
- หน้า **"โปรเจคทั้งหมด"** จะแสดง:
  - ชื่อและรายละเอียดโปรเจค
  - **เปอร์เซ็นต์ความสำเร็จ** (แถบ progress bar)
  - จำนวนงาน: **รอ** / **ทำอยู่** / **เสร็จ**

#### แก้ไข/ลบโปรเจค
- เข้าหน้าโปรเจค → กด **"แก้ไข"** เพื่อเปลี่ยนชื่อ/รายละเอียด
- กด **"ลบโปรเจค"** เพื่อลบ (จะลบ Task และ Sub-task ทั้งหมดด้วย)

---

### 2. จัดการ Task

#### เพิ่ม Task ใหม่
1. เข้าหน้าโปรเจค → กด **"+ เพิ่ม Task"**
2. กรอกข้อมูล:
   - **ชื่องาน** (จำเป็น)
   - **รายละเอียด**
   - **Deadline** (วันกำหนดส่ง)
   - **สถานะ** (รอดำเนินการ / กำลังทำ / เสร็จแล้ว)
   - **รองานจาก Task** (ถ้ามี)
   - **รองานจาก Sub-task** (ถ้ามี)
3. เลือกว่า **"มี Sub-task"** หรือไม่
   - ถ้ามี: กด **"+ เพิ่ม Sub-task"** เพื่อเพิ่มงานย่อยได้เลย
4. กด **"สร้าง Task"**

#### เปลี่ยนสถานะ Task
- ในหน้าโปรเจค → ใช้ **dropdown สถานะ** ข้าง Task → เลือก รอ/ทำอยู่/เสร็จ
- สถานะจะเปลี่ยนทันที

#### ลบ Task
- กด **"ลบ"** ข้าง Task (จะลบ Sub-task ทั้งหมดของ Task นี้ด้วย)

---

### 3. จัดการ Sub-task

#### เพิ่ม Sub-task
**วิธีที่ 1**: ตอนสร้าง Task ใหม่ → ติ๊ก "มี Sub-task" → เพิ่มได้ทันที

**วิธีที่ 2**: ในหน้าโปรเจค → กด **"+ เพิ่ม Sub-task"** ใต้ Task ที่ต้องการ
- กรอกชื่อ, รายละเอียด, deadline
- กด **"เพิ่ม"**

#### เปลี่ยนสถานะ/ลบ Sub-task
- เหมือนกับ Task → ใช้ dropdown หรือปุ่มลบ

---

### 4. การตั้งค่าการพึ่งพา (Dependency)

เมื่อ Task หรือ Sub-task ต้อง **รองานจาก** Task/Sub-task อื่น:

1. ตอนสร้างหรือแก้ไข Task → เลือกจาก dropdown:
   - **"รองานจาก Task"** → เลือก Task ที่ต้องรอ
   - **"รองานจาก Sub-task"** → เลือก Sub-task ที่ต้องรอ
2. ในหน้าโปรเจค จะแสดงข้อความ เช่น:
   - `รองานจาก Task: ออกแบบ Mockup`
   - `รองานจาก Sub-task: วิเคราะห์ข้อมูล`

## การตั้งค่า LINE Bot

### 1. เพิ่ม Bot เข้ากลุ่ม LINE

1. สแกน QR Code ของ Bot (จากหน้า LINE Developers) เพื่อเพิ่มเป็นเพื่อน
2. สร้าง **กลุ่ม LINE** (หรือใช้กลุ่มที่มีอยู่)
3. **เชิญ Bot เข้ากลุ่ม** → ระบบจะลงทะเบียนกลุ่มอัตโนมัติ

**หรือเพิ่มกลุ่มด้วยตนเอง:**
1. ไปที่เมนู **"ตั้งค่า LINE"** บน Web App
2. กรอก LINE Group ID และชื่อกลุ่ม → กด "เพิ่ม"

### 2. ตั้งค่าแจ้งเตือน

ในหน้า **"ตั้งค่า LINE"** → แต่ละกลุ่มสามารถเลือก:
- **ทุกวัน** - แจ้งเตือนทุกวัน
- **ทุก 3 วัน** - แจ้งเตือนทุก 3 วัน
- **ทุกอาทิตย์** - แจ้งเตือนสัปดาห์ละครั้ง

### 3. เลือกโปรเจคที่แจ้งเตือน

ในหน้า **"ตั้งค่า LINE"** → แต่ละกลุ่มจะแสดงโปรเจคทั้งหมด:
- กดที่ชื่อโปรเจค → **สีม่วง** = เปิดแจ้งเตือน, **สีเทา** = ปิด
- เลือกได้หลายโปรเจคต่อกลุ่ม
- แต่ละกลุ่มเลือกโปรเจคต่างกันได้

## การแจ้งเตือนผ่าน LINE

Bot จะส่งข้อความสรุปงานเข้ากลุ่ม LINE ตามความถี่ที่ตั้งค่า โดยมีรูปแบบ:

```
== สรุปงาน To-Do List ==

--- Website Redesign (33%) ---
Todo: 2 | Doing: 1 | Done: 1

[รอ] ออกแบบ Mockup
   Deadline: 15 มี.ค. 2569 (เหลือ 9 วัน)

[กำลังทำ] พัฒนา Frontend
   Deadline: 20 มี.ค. 2569 (เหลือ 14 วัน)
   รองานจาก Task: ออกแบบ Mockup
   [รอ] Header Component (เหลือ 12 วัน)
   [เสร็จ] Footer Component

[เสร็จ] ตั้งค่า Repository
```

ข้อมูลที่แสดง:
- **ชื่อโปรเจค** พร้อม **เปอร์เซ็นต์ความสำเร็จ**
- จำนวน Todo / Doing / Done
- แต่ละ Task พร้อม **สถานะ**, **Deadline**, และ **จำนวนวันที่เหลือ**
- หากเลยกำหนด จะแสดง เช่น `เลยกำหนด 3 วัน`
- Sub-task แสดงเยื้องใต้ Task
- แสดง **"รองานจาก..."** หาก Task มี dependency

## คำอธิบายสถานะ

| สถานะ | ความหมาย | สีบน Web | ในข้อความ LINE |
|--------|---------|---------|---------------|
| **Todo** | รอดำเนินการ ยังไม่ได้เริ่ม | สีเทา | [รอ] |
| **Doing** | กำลังทำอยู่ | สีฟ้า | [กำลังทำ] |
| **Done** | เสร็จเรียบร้อย | สีเขียว | [เสร็จ] |

### การคำนวณเปอร์เซ็นต์ความสำเร็จ
- นับ Task และ Sub-task **ทั้งหมด** ในโปรเจค
- `% = (จำนวน Done / จำนวนทั้งหมด) x 100`
- เช่น มี 10 items, Done 3 = **30%**

### การนับวันที่เหลือ
- **เหลือ 5 วัน** = ยังไม่ถึง deadline
- **ครบกำหนดวันนี้** = deadline คือวันนี้
- **เลยกำหนด 3 วัน** = เลย deadline ไป 3 วัน (ติดลบ)
